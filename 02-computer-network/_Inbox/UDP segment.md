# UDP Segment (UDP 데이터그램)

## 개요
UDP Segment는 UDP에서 전송되는 데이터 단위로, 일반적으로 **UDP 데이터그램(datagram)**이라고도 불린다.  

TCP와 달리 연결 상태를 유지하지 않으며, 각 세그먼트는 **독립적으로 전송**된다.  
구조가 단순하고 헤더 크기가 작아 빠른 데이터 전달이 가능하다.

---

## 구조
UDP 세그먼트는 **헤더(Header) + 데이터(Data)**로 구성된다.

### 1. UDP 헤더 (8 bytes)
- 총 8바이트로 매우 간단한 구조

| 필드 | 크기 | 설명 |
|------|------|------|
| Source Port | 16 bit | 송신 포트 번호 |
| Destination Port | 16 bit | 수신 포트 번호 |
| Length | 16 bit | 전체 길이 (헤더 + 데이터) |
| Checksum | 16 bit | 오류 검출 |

---

### 2. 데이터 (Payload)
- 실제 전송되는 애플리케이션 데이터
- 크기 제한: 이론적으로 최대 65,507 bytes (IP 헤더 제외)

---

## 특징
- **고정된 헤더 크기 (8 bytes)** → 오버헤드 최소화  
- **순서 정보 없음** → 패킷 순서 보장 X  
- **재전송 기능 없음** → 손실 발생 가능  
- **각 세그먼트 독립 처리** → 연결 상태 필요 없음  

---

## 동작 방식
1. 애플리케이션이 데이터를 생성
2. UDP가 헤더를 붙여 세그먼트 생성
3. IP 계층으로 전달되어 네트워크로 전송
4. 수신 측에서 포트 번호 기반으로 애플리케이션 전달

---

## TCP Segment와 비교

| 항목 | UDP Segment | TCP Segment |
|------|------------|------------|
| 헤더 크기 | 8 bytes | 20 bytes 이상 |
| 신뢰성 | 없음 | 있음 |
| 순서 제어 | 없음 | 있음 |
| 흐름 제어 | 없음 | 있음 |
| 용도 | 실시간 전송 | 신뢰성 요구 전송 |

---

## 정리
UDP Segment는 **간단한 구조와 빠른 전송을 위한 최소한의 정보만 포함한 데이터 단위**이다.  
신뢰성 대신 속도를 선택한 설계로, 실시간 서비스에 적합하다.
