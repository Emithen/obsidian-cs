# UDP (User Datagram Protocol)

## 개요
UDP(User Datagram Protocol)는 전송 계층(Transport Layer)에서 동작하는 **비연결형(connectionless) 프로토콜**이다.  
데이터를 빠르게 전송하는 데 초점을 맞추며, 신뢰성보다는 **속도와 효율성**을 우선시한다.

TCP와 달리 연결 설정 과정(3-way handshake)이 없고, 데이터 전송 후에도 확인 응답(ACK)을 받지 않는다.  
이로 인해 패킷 손실, 순서 뒤바뀜 등이 발생할 수 있지만, 오버헤드가 적어 실시간 통신에 적합하다.

---

## 특징
- **비연결형 프로토콜**
  - 송신자와 수신자 간 사전 연결 없이 데이터 전송

- **신뢰성 보장 없음**
  - 패킷 손실, 중복, 순서 변경 가능

- **빠른 전송 속도**
  - 연결 설정 및 제어 과정이 없어 지연이 적음

- **헤더 크기 작음 (8바이트)**
  - TCP(20바이트 이상)보다 효율적

---

## 동작 방식
1. 송신자가 데이터그램(datagram)을 생성
2. IP 계층을 통해 목적지로 전달
3. 수신자는 도착한 데이터그램을 그대로 처리

※ 중간에 손실되거나 순서가 바뀌어도 재전송하지 않음

---

## 사용 사례
- 실시간 스트리밍 (영상, 음성)
- 온라인 게임
- DNS 질의
- VoIP (인터넷 전화)

---

## 장점
- 낮은 지연(latency)
- 구현 단순
- 네트워크 자원 절약

---

## 단점
- 데이터 손실 가능성
- 순서 보장 없음
- 흐름 제어 및 혼잡 제어 없음

---

## TCP와 비교

| 항목 | UDP | TCP |
|------|-----|-----|
| 연결 방식 | 비연결형 | 연결형 |
| 신뢰성 | 없음 | 있음 |
| 속도 | 빠름 | 상대적으로 느림 |
| 순서 보장 | 없음 | 있음 |
| 용도 | 실시간 서비스 | 파일 전송, 웹 |

---

## 정리
UDP는 **빠르고 가벼운 데이터 전송**이 필요한 환경에서 사용되는 프로토콜이다.  
신뢰성이 필요한 경우에는 TCP를, **속도와 실시간성이 중요한 경우에는 UDP를 선택**하는 것이 일반적이다.
