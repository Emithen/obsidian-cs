# One's Complement Sum (1의 보수 덧셈)

## 개요

One's Complement Addition은 **이진수 덧셈 방식 중 하나**로,
주로 네트워크 프로토콜(예: UDP, IP)의 체크섬 계산에 사용된다.

일반적인 이진 덧셈과 유사하지만, **오버플로우(자리올림)가 발생하면 이를
다시 하위 비트에 더하는 특징**이 있다.

------------------------------------------------------------------------

## 핵심 개념

-   모든 연산은 **16비트 단위**로 수행
-   덧셈 결과에서 발생한 **캐리(carry)를 다시 더함 (end-around carry)**
-   최종 결과에 **1의 보수(비트 반전)**를 취해 체크섬 생성

------------------------------------------------------------------------

## 동작 방식

1.  두 이진수를 더함
2.  최상위 비트에서 캐리 발생 시:
    -   해당 값을 결과의 하위 비트에 더함
3.  결과를 반복적으로 정리
4.  필요 시 1의 보수를 취함

------------------------------------------------------------------------

## 예시

### Step 1: 일반 덧셈

1111 0000 0000 0001 + 0001 0000 0000 0001 -------------------- 1 0000
0000 0000 0010

### Step 2: 캐리 처리 (end-around carry)

0000 0000 0000 0010 + 1 -------------------- 0000 0000 0000 0011

------------------------------------------------------------------------

## 특징

-   **오류 검출에 적합**
-   하드웨어 구현이 간단
-   순서에 관계없이 동일 결과 (교환법칙 성립)
-   네트워크 프로토콜에서 널리 사용됨

------------------------------------------------------------------------

## 사용 사례

-   UDP Checksum
-   IP Header Checksum
-   TCP Checksum

------------------------------------------------------------------------

## 일반 이진 덧셈과 차이

  항목        일반 이진 덧셈   1의 보수 덧셈
  ----------- ---------------- ---------------
  캐리 처리   버림             다시 더함
  사용 목적   일반 연산        오류 검출
  활용 분야   범용             네트워크

------------------------------------------------------------------------

## 정리

One's Complement Addition은 **캐리를 순환시켜 더하는 독특한 덧셈
방식**으로,\
간단하면서도 효과적인 오류 검출을 가능하게 하여 네트워크 프로토콜에서
핵심적으로 사용된다.
