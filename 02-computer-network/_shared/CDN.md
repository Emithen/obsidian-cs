# CDN (Content Delivery Network)

## 개요
CDN(Content Delivery Network)은 전 세계 여러 지역에 분산된 서버 네트워크를 통해 사용자에게 콘텐츠를 빠르고 안정적으로 전달하는 시스템이다.  
사용자가 콘텐츠를 요청하면, 물리적으로 가장 가까운 서버(엣지 서버)에서 데이터를 제공하여 지연 시간을 줄이고 성능을 향상시킨다.

## 주요 목적
- **지연 시간 감소 (Latency 감소)**: 사용자와 가까운 서버에서 콘텐츠 제공
- **로드 분산 (Load Balancing)**: 트래픽을 여러 서버에 분산
- **가용성 향상 (High Availability)**: 장애 발생 시 다른 서버로 자동 우회
- **보안 강화**: DDoS 방어, SSL/TLS 암호화 지원

## 동작 방식
1. 사용자가 웹사이트에 접속
2. DNS가 CDN 서버로 요청을 라우팅
3. 가장 가까운 엣지 서버에서 콘텐츠 제공
4. 캐시에 없을 경우 원본 서버(Origin)에서 가져와 저장 후 전달

## 주요 구성 요소
- **Origin Server**: 원본 콘텐츠를 보관하는 서버
- **Edge Server (PoP)**: 사용자에게 콘텐츠를 전달하는 CDN 서버
- **Cache**: 자주 요청되는 콘텐츠를 저장하는 공간
- **DNS Routing**: 사용자 요청을 최적의 서버로 연결

## 장점
- 빠른 콘텐츠 로딩 속도
- 서버 부하 감소
- 글로벌 서비스 품질 향상
- 보안 기능 제공

## 단점
- 비용 발생
- 캐시 무효화(Cache Invalidation) 관리 필요
- 동적 콘텐츠 처리의 한계 (일부 해결 기술 존재)

## 사용 사례
- 웹사이트 정적 파일 제공 (이미지, CSS, JS)
- 스트리밍 서비스 (영상, 음악)
- 게임 패치 및 대용량 파일 배포
- API 응답 캐싱
