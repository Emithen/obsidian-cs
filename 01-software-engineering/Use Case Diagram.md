
## 개요

Use Case Diagram은 UML에서 사용되는 다이어그램으로,\
**시스템과 외부 액터(사용자 또는 외부 시스템) 간의 상호작용을 기능
중심으로 표현**한다.

이 다이어그램은 시스템 내부 구조가 아니라,\
👉 *사용자가 시스템을 통해 수행할 수 있는 행동(Use Case)* 을 중심으로
나타낸다.

### 핵심 특징

-   사용자 관점에서 **기능 요구사항을 정의**
-   시스템의 **범위(System Boundary)** 를 명확히 표현
-   액터(Actor)와 유스케이스 간 **관계 중심 모델링**
-   비개발자도 이해하기 쉬운 **직관적인 표현 방식**

### 주요 요소

-   **Actor**: 시스템과 상호작용하는 외부 주체 (사용자, 외부 시스템 등)
-   **Use Case**: 시스템이 제공하는 기능 (예: 로그인, 결제)
-   **System Boundary**: 시스템의 범위를 나타내는 경계
-   **Relationship**: 요소 간 관계 (Association, Include, Extend,
    Generalization)

### 사용 목적

-   요구사항 분석 단계에서 기능을 식별
-   시스템의 기능 범위를 정의
-   이해관계자와의 커뮤니케이션 도구로 활용
