## Week5 - 115 page

- L2 norm
	- 결과 값의 위상을 표현하는 한 방식
	- 기하학적으로는 벡터의 스칼라 값의 제곱이다
- x 벡터
	- x 의 길이 == feature 개수
- W 행렬
	- 열 개수 == input dim
	- 행 개수 == output dim
- input feature, output feature 의 역할
	- input feature
		- 데이터가 원래 가지고 있던 속성
		- ex) 픽셀 값, 몸무게, 키 등등
	- output feature
		- 새롭게 부여한 속성
		- ex) 숫자 모양, 곡선이 얼마나 있는지, 세로 선이 얼마나 있는지
- FC - fully connected layer
	- input <=> output 모든 경우의 수에 대해 연결을 생성
- LC - locally connected layer
	- output 마다 input 의 전체 데이터 요소를 바탕으로 output 을 생성하는 것이 아닌 일부만 사용
- Convolution
	- x 의 모든 input feature 에 대해서 열을 갖는 W 가 아닌 작은 W 를 지정
	- 만들어진 작은 W (kernel) 를 하나의 x 에 대해서 windowing 을 함
	- ex)
		- x - 200 x 200 pixel
		- W - 200 x 200 x 40000
		- Convolution W - 10 x 10 x 40000
	- 설정
		- padding
		- stride
		- dilation
	- 장점 - 이미지 객체 판정 예시
		- 사람 얼굴을 판별하는 모델을 학습시키기 위해서는 다양한 사람 얼굴 이미지를 output feature 로 갖는 W 를 준비해야 한다
		- 자연스럽게 W 행 개수가 매우 많아진다
			- 사람 얼굴 이미지는 종류가 너무 다양하기 때문 (각도, 화면 내 정렬 위치, 얼굴형 등등)
		- convolution 의 경우
			- 눈, 코, 입 필터 등을 따로 준비한다
			- 화면 내의 정렬 위치가 달라져도 눈 필터는 특정 좌표에서 높은 유사도를 뱉어준다
			- 코, 입도 마찬가지
			- 사람 얼굴을 몇 가지 구성 요소로 나눌 수 있다면 해당 구성 요소의 지역적 배치 조합 경우의 수를 걱정할 필요 없이 위치에 관계없는 존재성 판정을 할 수 있게 된다
- Pooling layer
	- 쉽게 말하자면 화질 열화 같은 개념
	- 커널을 갖다 대고 output dimension 을 줄여준다
	- feature 개수를 줄이는 효과
	- 보통 하나의 layer 로 잘 쳐주지 않는다
- Softmax
	- max 함수는 미분 불가능
		- 애초에 불연속이다
	- 그래서 각 값에 exp 를 씌워서 편차를 늘림
	- 각 값이 전체 값의 합에서 차지하는 비중을 계산해서 반환 -> 0~1 까지의 확률로 취급 가능한 값이 나옴
- Cross Entrophy
	- - y * log(p)
		- 1 에 가까울 수로 완만해짐
		- 0 에 가까울 수록 가파르고 커짐
		- 틀렸을 때의 페널티 크다
		- 유사하게 맞췄을 때 페널티 작다
- Activation Function
	- 비선형 결합을 제공한다
	- 앞 뒤 layer 의 선형 병합을 막는다
